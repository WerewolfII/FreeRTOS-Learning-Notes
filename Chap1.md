# ✅ 第1章：为什么需要RTOS（裸机 vs RTOS）

（所有FreeRTOS学习最核心一章）

---

## 📌 一、核心知识点

### 1️⃣ 裸机编程模型（Super Loop）

典型STM32程序：

```c
while(1)
{
    LED_Task();
    Key_Task();
    UART_Task();
}
```

特点：

* 顺序执行
* 无优先级
* 无调度
* 时间不可预测

---

### 2️⃣ 裸机系统的典型问题

#### ❌ 响应慢

```
UART处理期间 → LED无法响应
```

---

#### ❌ 强耦合

```
一个模块影响全部系统
```

---

#### ❌ 延时阻塞CPU

```c
delay_ms(1000);
```

CPU浪费1000ms。

---

#### ❌ 无法扩展

项目变大会变：

```
超级大while循环
```

（你应该见过）

---

### 3️⃣ RTOS解决什么问题

RTOS提供：

| 能力   | 作用     |
| ---- | ------ |
| 多任务  | 并行执行   |
| 调度器  | 自动切换   |
| 优先级  | 关键任务优先 |
| 通信机制 | 安全数据交换 |
| 资源管理 | 避免冲突   |

---

### 4️⃣ RTOS核心思想（最重要）

RTOS本质：

```
CPU时间分配管理器
```

它决定：

```
谁什么时候运行多久
```

---

### 5️⃣ STM32 + FreeRTOS运行模式

```
main()
  ↓
创建任务
  ↓
启动调度器
  ↓
永远不返回
```

之后：

```
你不再控制CPU
RTOS控制CPU
```

这是最大思维转变。

---

## 🧠 二、原理深入（工程师必须理解）

---

### ⭐ RTOS不是并行执行

MCU单核：

```
快速切换任务
```

产生：

```
并发假象
```

---

### ⭐ CPU时间切片

```
TaskA → TaskB → TaskC → TaskA
```

切换速度：

```
微秒级
```

---

### ⭐ 上下文切换（核心）

RTOS做：

```
保存寄存器
恢复寄存器
```

STM32上：

```
硬件 + PendSV实现
```

（后面会讲）

## 💻 三、STM32F103实例（裸机 vs RTOS对比）

---

### 🔴 裸机实现（阻塞）

```c
while(1)
{
    LED_Toggle();
    delay_ms(500);

    UART_Send("Hello");
}
```

问题：

* UART会卡LED
* CPU利用率低

---

### 🟢 FreeRTOS实现

```c
void LedTask(void *arg)
{
    while(1)
    {
        LED_Toggle();
        vTaskDelay(500);
    }
}

void UartTask(void *arg)
{
    while(1)
    {
        UART_Send("Hello");
        vTaskDelay(1000);
    }
}
```

效果：

* 同时执行
* 不阻塞
* 自动调度

## ⚡ 四、工程经验（真实项目结论）

当满足以下任意一条：

✅ 多模块
✅ 实时要求
✅ 通信复杂
✅ GUI
✅ 网络协议
✅ 工业控制

→ 必须RTOS。

---

### STM32项目分水岭

| 项目规模  | 建议   |
| ----- | ---- |
| <5k行  | 裸机   |
| >10k行 | RTOS |
| 工业产品  | RTOS |
