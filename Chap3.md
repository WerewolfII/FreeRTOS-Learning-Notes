# 📌 一、核心知识点

---

## 1️⃣ 什么是并发（Concurrency）

---

### ✅ 定义

```text
多个任务在同一时间段内推进
```

注意：

```text
不一定同时执行
```

---

### 举例（STM32）

CPU执行顺序：

```
LED任务 → UART任务 → 传感器任务 → LED任务
```

从外部看：

```
三个任务在同时运行
```

这就是并发。

---

### ⭐ 本质

```text
时间交错执行
```

---

---

## 2️⃣ 并发 vs 并行（面试必问）

---

### 🟦 并发（Concurrency）

| 特点     | 说明 |
| ------ | -- |
| 单核可实现  | ✅  |
| 交替执行   | ✅  |
| RTOS核心 | ✅  |

---

### 🟥 并行（Parallelism）

| 特点        | 说明  |
| --------- | --- |
| 多核CPU     | 必须  |
| 真同时执行     | 是   |
| STM32F103 | 不支持 |

---

### STM32F103情况

基于：

* ARM Cortex-M3 单核

所以：

```text
只有并发，没有并行
```

---

### ⭐ 图示理解

---

#### 并发（单核）

```
TaskA → TaskB → TaskC → TaskA
```

---

#### 并行（多核）

```
Core1: TaskA
Core2: TaskB
Core3: TaskC
```

---

---

## 3️⃣ 为什么需要多任务模型

---

### 裸机问题

```c
while(1)
{
    LED();
    UART();
    SENSOR();
}
```

问题：

* 无优先级
* 强耦合
* 难扩展

---

---

### RTOS模型

```
Task_LED
Task_UART
Task_SENSOR
```

RTOS负责：

```
调度执行顺序
```

---

### ⭐ 最大区别

| 裸机    | RTOS |
| ----- | ---- |
| 函数    | 任务   |
| 顺序执行  | 调度执行 |
| 无状态管理 | 有状态机 |

---

---

## 4️⃣ 什么是任务（Task）

---

### ✅ 定义

```text
可被调度执行的独立程序单元
```

---

### 每个任务拥有：

| 属性  | 作用             |
| --- | -------------- |
| 独立栈 | 保存局部变量         |
| 优先级 | 决定执行顺序         |
| 状态  | Ready/Blocked等 |
| 上下文 | CPU寄存器状态       |

---

---

### ⭐ 任务 ≈ 线程（但更轻量）

RTOS任务类似：

```
PC线程
```

但：

```
更简单
更可控
```

---

---

## 5️⃣ 多任务执行本质

（极重要）

---

### RTOS做两件事：

---

### ① 保存当前任务状态

```
保存寄存器
保存栈
```

---

### ② 恢复另一个任务

```
恢复寄存器
恢复栈
```

---

这叫：

```text
上下文切换（Context Switch）
```

---

---

## 6️⃣ 时间片（Time Slice）

---

### 概念

```text
每个任务执行一小段时间
```

例如：

```
1ms给TaskA
1ms给TaskB
1ms给TaskC
```

循环。

---

### 效果

```
看起来同时运行
```

---

---

## 7️⃣ 多任务系统结构（RTOS核心结构）

---

### RTOS内部：

```
Scheduler（调度器）
    ↓
选择最高优先级任务
```

---

### 执行流程

```
任务A运行
    ↓
时间到
    ↓
切换任务B
```

# 🧠 二、原理深入（工程师必须理解）

---

## ⭐ MCU如何实现多任务？

---

### 单核CPU限制

STM32F103：

```
同一时间只能执行一条指令
```

---

### RTOS解决方案

```
极快切换任务
```

速度：

```
微秒级
```

人感觉：

```
同时执行
```

---

---

## ⭐ 上下文切换原理（简化版）

---

### 假设：

```
TaskA执行到一半
```

RTOS做：

```
1 保存R0-R15寄存器
2 保存SP
3 切换栈指针
4 恢复TaskB寄存器
```

---

### 切换时间：

```
约几微秒
```

---

---

## ⭐ 为什么每个任务必须有独立栈？

---

### 因为：

```
函数调用需要栈
局部变量在栈上
```

如果共享：

```
数据混乱
```

---

### 所以：

```
每任务独立栈
```

这是RTOS核心设计。

---

---

## ⭐ 多任务不是免费的（重要工程认知）

---

### 代价：

| 成本    | 原因     |
| ----- | ------ |
| RAM增加 | 每任务独立栈 |
| CPU开销 | 上下文切换  |
| 调度复杂度 | 优先级管理  |

---

### STM32F103常见情况

```
RAM只有20KB
```

任务数不能乱开。

---

# 💻 三、STM32F103实例（最小多任务）

---

## 示例：LED + UART并发

---

### 创建两个任务

```c
void LedTask(void *arg)
{
    while(1)
    {
        LED_Toggle();
        vTaskDelay(500);
    }
}

void UartTask(void *arg)
{
    while(1)
    {
        printf("Hello\n");
        vTaskDelay(1000);
    }
}
```

---

### 创建任务

```c
xTaskCreate(LedTask, "LED", 128, NULL, 1, NULL);
xTaskCreate(UartTask, "UART", 128, NULL, 1, NULL);
```

---

### 启动调度器

```c
vTaskStartScheduler();
```

---

### 结果

```
LED闪烁
UART打印
同时进行
```
