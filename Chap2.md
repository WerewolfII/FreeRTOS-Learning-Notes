# ✅ 第2章：什么是实时系统（Hard / Soft Real-Time）

（这是理解 FreeRTOS 能力边界最关键的一章）

这一章会改变一个常见误区：

```text
实时 ≠ 快
```

而是：

```text
实时 = 可预测
```

---

# 📌 一、核心知识点

---

## 1️⃣ 什么是实时系统（Real-Time System）

### ✅ 定义（工程标准）

实时系统：

> **在规定时间内必须完成任务的系统。**

关键不是速度，而是：

```
能否在Deadline前完成
```

---

### ⭐ 核心概念：Deadline（截止时间）

例如：

| 任务     | 必须完成时间 |
| ------ | ------ |
| 电机控制   | 1ms内   |
| 飞控姿态计算 | 2ms内   |
| 传感器采样  | 100μs内 |

只要超过：

```
系统算失败
```

即使只超1μs。

---

## 2️⃣ 实时 ≠ 快（非常重要）

---

### ❌ 错误理解

```text
STM32很快 → 就是实时
```

错。

---

### ✅ 正确理解

```text
响应时间稳定可预测 → 才是实时
```

---

### 举例

#### 非实时系统（PC）

| 执行时间  |
| ----- |
| 1ms   |
| 10ms  |
| 200ms |
| 不确定   |

→ 不可预测。

---

#### 实时系统（嵌入式）

| 执行时间   |
| ------ |
| 1.01ms |
| 1.02ms |
| 1.00ms |

→ 可预测。

---

### ⭐ 工程结论

```
稳定比快更重要
```

---

## 3️⃣ 实时系统分类

---

### 🟥 Hard Real-Time（硬实时）

---

### 定义

```
错过deadline = 系统失败
```

---

### 典型系统

| 场景    | 后果 |
| ----- | -- |
| 飞机飞控  | 坠机 |
| 汽车ABS | 失控 |
| 工业机器人 | 撞人 |
| 医疗设备  | 危险 |

---

### 特点

* 绝对确定性
* 必须时间分析
* 通常不用动态内存
* 必须Worst-case分析

---

### ⭐ FreeRTOS能否Hard Real-Time？

理论：

```
可接近，但不保证
```

原因：

* 中断影响
* Cache影响
* 调度开销
* 动态内存

后面会讲。

---

---

### 🟨 Soft Real-Time（软实时）

---

### 定义

```
偶尔超时可接受
```

---

### 典型系统

| 系统    | 允许延迟 |
| ----- | ---- |
| GUI界面 | 可以   |
| 音频播放  | 可少量  |
| 网络通信  | 可以   |
| IoT设备 | 可以   |

---

### STM32 + FreeRTOS通常属于

```
Soft Real-Time
```

---

---

### 🟦 Firm Real-Time（工程中常见）

---

介于两者之间：

```
偶尔超时 → 数据无效
但系统不崩
```

例：

* 摄像头采样
* 雷达数据

---

## 4️⃣ 嵌入式为什么需要实时性

---

### STM32典型任务

| 任务     | 是否实时 |
| ------ | ---- |
| 电机PWM  | 强实时  |
| UART收发 | 中实时  |
| LED闪烁  | 不重要  |
| 按键扫描   | 中    |

---

RTOS允许：

```
重要任务优先
```

---

## 5️⃣ 实时性的数学表达（工程理解）

---

### ⭐ 最坏执行时间（WCET）

```
Worst Case Execution Time
```

---

RTOS设计目标：

```
执行时间 < Deadline
```

---

### 举例

```
采样任务执行 200μs
deadline 500μs
```

安全。

---

如果：

```
最大可能 600μs
```

→ 不实时。

---

---

## 6️⃣ STM32 + FreeRTOS实时性来源

（工程必须理解）

---

### ① 抢占式调度

高优先级可打断低优先级。

---

### ② 中断机制

STM32基于：

* ARM Cortex-M3
* NVIC硬件调度

中断延迟：

```
约12个CPU周期
```

极低。

---

---

### ③ 优先级机制

```
关键任务永远先执行
```

---

### ④ Tick节拍

保证时间基准。

---

---

## 7️⃣ 实时性 vs 吞吐量（工程权衡）

---

### 两种设计目标

| 类型   | 目标   |
| ---- | ---- |
| 实时系统 | 时间确定 |
| PC系统 | 吞吐最大 |

---

RTOS选择：

```
确定性 > 性能
```

# 🧠 二、原理深入（面试必问）

---

## ⭐ RTOS如何保证实时性？

---

### 1️⃣ 可预测调度

调度算法固定：

```
优先级调度
```

---

### 2️⃣ 固定时间内核操作

FreeRTOS API设计：

```
O(1)操作
```

（常数时间）

---

### 3️⃣ 中断延迟可计算

在 STM32F103 上：

```
中断延迟基本固定
```

---

---

## ⭐ 为什么Linux不适合硬实时？

（对比理解）

---

Linux：

* 调度复杂
* 内存分页
* 不确定延迟

---

FreeRTOS：

```
简单 → 可预测
```

# 💻 三、STM32F103实例（实时 vs 非实时）

---

## 🔴 非实时实现（裸机）

```c
void Sensor_Read()
{
    delay_ms(10);  // 不可预测
}
```

问题：

```
延迟变化大
```

---

---

## 🟢 实时实现（RTOS任务）

```c
void SensorTask(void *arg)
{
    while(1)
    {
        ReadSensor();
        vTaskDelayUntil(&lastWake, 1); // 周期执行
    }
}
```

特点：

```
固定周期执行
```

（实时系统核心）
